Starting myFamiPedia locally

Five terminal windows, run in order. All from C:\Users\leach\myfamipedia unless noted.

0. Docker

Check first — no terminal needed if already running

Confirm Postgres and Redis containers are up (matching DATABASE_URL / REDIS_URL in .env).

1. Database migrations + seed data run once / after schema changes

Terminal 1
cd C:\Users\leach\myfamipedia
npx pnpm@9.15.9 migrate
npx pnpm@9.15.9 seed

Migrate is only needed the first time, or after a migration file is added. Seed loads reference data (e.g. the curated interview question bank) — separate from migrations, easy to forget, safe to re-run. Can close this terminal once both finish.

2. API server keep open

Terminal 2
cd C:\Users\leach\myfamipedia
npx pnpm@9.15.9 dev:api

3. Background workers keep open

Terminal 3
cd C:\Users\leach\myfamipedia
npx pnpm@9.15.9 workers

Handles async jobs (transcription, notifications, scheduled prompts, etc.). Optional if you're only testing screens that don't touch those jobs, but leave it running normally.

4. Web app keep open

Terminal 4
cd C:\Users\leach\myfamipedia
npx pnpm@9.15.9 dev:web

Vite will print the local URL (usually http://localhost:5173).

5. Mobile app keep open

Terminal 5 — run from apps\mobile
cd C:\Users\leach\myfamipedia\apps\mobile
npx expo start -c

The -c clears Expo/Metro's bundler cache — always use it after dependency changes or when a fix doesn't seem to take effect. Scan the QR code with Expo Go, or press a / i for an emulator.

Shutting down

Ctrl+C in each terminal (4, 3, 2), then stop the Docker containers if you want. Terminal 1 (migrations) doesn't need stopping — it already exited on its own.

One-time setup note

pnpm isn't on this machine's PATH directly — all commands above go through npx pnpm@9.15.9. To skip that prefix everywhere (including inside expo install, which invokes pnpm directly and fails without it), run npm install -g pnpm@9.15.9 once.
