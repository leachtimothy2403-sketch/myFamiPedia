## Real bugs found and fixed this session

These weren't invented scope — they were discovered by actually building the frontend against the real API and hitting things that didn't work:

1. **`persons_tree_view` was missing `name`, `birth_date`, `death_date`** (and a few other columns) — it only ever exposed `id`, `family_group_id`, `status`, and two privacy-masked fields. Every endpoint that reads through this view (tree, profile, summary) was structurally unable to return a person's name. No test caught it because nothing asserted on `.name`. Fixed with migration `019_persons_tree_view_core_fields.js`. **Tim needs to run `npx pnpm migrate` if he hasn't already** — confirmed done earlier in this session, but flagging in case this doc is read out of order.
2. **`packages/shared`'s compiled output was CommonJS**, which Vite/Rollup couldn't statically resolve named exports from (`ApiClient` import failed at build time). Switched the package to native ESM output (`tsconfig.json` module target + `package.json` `"type": "module"`). Confirmed `apps/api` never imports this package at runtime, so it was safe; `apps/mobile` does import it and this hasn't caused problems, but worth a look if mobile bundling ever acts up.
3. **API returns snake_case, shared types are camelCase** — added a client-side transform (`packages/shared/src/lib/caseTransform.ts`) applied in `ApiClient.parse()`, since the API contract itself is tested and snake_case (not something to change).
4. **`apiClient.addFamilyMember` posted to `POST /persons`, which doesn't exist.** Replaced with `inviteFamilyMember` (→ `POST /invitations`, living branch) and `addDeceasedProfile` (→ `POST /persons/deceased`, Section 4 branch), matching the routes that actually exist.
5. **No general memory-creation endpoint existed at all** — only the deceased-profile posthumous route did. Added `POST /memories` to `apps/api/src/routes/memories.routes.ts`. Verified against pglite (creates correctly, shows in feed + timeline, tagging another person via `personIds` surfaces it on their profile).
6. **Both web's and mobile's tree screens called `getFamilyTree("me")`** — not a real family group id. Fixed by decoding `personId`/`familyGroupId` out of the JWT client-side (no `/auth/me` endpoint exists, and login doesn't echo the person back the way register does). Web: `apps/web/src/lib/session.ts`. Mobile: `apps/mobile/lib/session.ts` (needed a hand-rolled base64 decoder since Hermes has no `atob`).
7. **`apps/mobile` had no `@types/node`**, so `lib/apiClient.ts`'s `process.env` usage could never have typechecked cleanly. Fixed, but this caused a process mistake — see below.

## Process notes for whoever picks this up next (probably me, but worth writing down)

1. **This Cowork sandbox's Linux shell has an unreliable view of the mounted repo folder.** Reads shortly after writes are sometimes stale or truncated (confirmed repeatedly — files come back missing content or don't exist at all, even though the same file read via the `Read` tool, which goes through a different path, is correct). When verifying code in a scratch copy, don't trust `rsync`/`cat` from the mount for recently-edited files — re-`Read` them and `heredoc` the known-good content directly into the scratch copy instead.
2. **Never run `git` commands from this sandbox's shell against the real repo.** It reliably creates lock/temp files it then can't delete (`.git/index.lock`, `.git/objects/*/tmp_obj_*`), which then blocks Tim's own local git too until he manually deletes them. All git operations (`add`/`commit`/`push`) should be commands handed to Tim to run himself in PowerShell.
3. **Whenever a `package.json` gets a new dependency, have Tim run `npx pnpm install` (not `--frozen-lockfile`) and commit the updated `pnpm-lock.yaml` in the *same* commit as the `package.json` change**, before pushing. This session shipped a commit with `apps/mobile/package.json`'s new `@types/node` but a stale lockfile, which broke CI's `pnpm install --frozen-lockfile` step (fails in ~10s, looks like a real failure but isn't). Caught and fixed same-day, but avoidable.
4. **Verification workflow that works:** copy the relevant `apps/*` + `packages/shared` + root config files to `/tmp/<scratch-name>` (real Linux disk, not the FUSE mount), `npx pnpm@9.15.9 install --no-frozen-lockfile --filter "./packages/shared" --filter "./apps/X"`, then `tsc -b`/`tsc --noEmit` and `vite build` or `vitest run` as appropriate. Always check real exit codes (`echo $?` immediately after the command, never through a pipe — `| tail` swallows the real exit status and will silently report false passes).
5. Tim's local dev environment specifics (Docker port remap, `npx pnpm` instead of bare `pnpm`, etc.) are unchanged from the prior handover — see that doc or `docs/handover` history for the original details if this file doesn't cover it.
